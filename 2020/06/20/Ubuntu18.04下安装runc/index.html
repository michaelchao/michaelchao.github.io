Ubuntu18.04下安装runc

Posted on 2020-06-20

本文介绍如何编译go源码，然后用生成的go程序编译runc

go1.4之后的版本需要用go程序去编译，那这个go程序就称之为bootstrap

  1. go1.4解压到/home/ts/go1.4下(此目录下有api AUTHORS bin等文件和目录)
  2. cd src(上面一群目录中有个src)
  3. GOOS=linux GOARCH=amd64 CGO_ENABLED=0 ./make.bash
  4. ~/.profile末尾添加export PATH=”$PATH:/home/ts/go1.4/bin”
  5. 退出当前终端

用上面生成好的bootstrap去编译可以用来编译runc的go版本(本文使用go1.6)

  1. 重新登录终端
  2. source ~/.profile
  3. go1.6解压到/home/ts/go_src/go_1_6/go下(此目录下有api AUTHORS bin等文件和目录)
  4. cd src(上面一群目录中有个src)
  5. ./ make.bash
  6. ~/.profile末尾删除export PATH=”$PATH:/home/ts/go1.4/bin”
  7. ~/.profile末尾添加export PATH=”$PATH:/home/ts/go_src/go_1_6/go/bin”
  8. 退出当前终端

用go1.6去编译runc

  1. 重新登录终端
  2. source ~/.profile
  3. cd 至目录/home/ts/go
  4. export GOPATH=/home/ts/go
  5. mkdir -p src/github.com/opencontainers
  6. cd src/github.com/opencontainers
  7. git clone https://github.com/opencontainers/runc
  8. git checkout v1.0.0-rc5
  9. make
  10. runc可执行程序在当前目录生成

运行镜像

  1. 从https://download.openvz.org/template/precreated/网址下载一个镜像
  2. 创建目录/tmp/mycontainer
  3. 将镜像解压到目录/tmp/mycontainer并命名为rootfs（这个目录下有一般的linux系统目录）
  4. 运行sudo runc spec创建config.json文件
  5. 修改此文件中的”terminal”: true为”terminal”: false
  6. 修改此文件中的”args”:[“sh”]为”args”:[“sleep”， “3000”]
  7. sudo runc create mybusybox 创建容器
  8. sudo runc create list 查看当前运行信息
  9. sudo runc create state mybusybox查看容器状态
  10. sudo runc create start mybusybox开始运行容器
  11. sudo runc exec mybusybox ls在容器中执行命令ls
